# 📚 论文检索助手 — 新手教程

## 这是什么东西？

一个**带图形界面的论文搜索工具**。你在搜索框里输入关键词，程序帮你从 ArXiv（全球最大的免费学术论文网站）搜索相关论文，然后用 AI 帮你做两件事：

1. **读论文** → 用中文总结论文讲了什么（不用啃英文了）
2. **翻译摘要** → 把英文摘要翻译成中文

---

## 三个工具是怎么工作的？

### 🔍 工具1：search_arxiv — 搜索论文

```
你输入 "RAG" → 程序访问 ArXiv API → 返回相关论文列表
```

**通俗理解：** 就像你在 Google 搜索，但专门搜学术论文。程序把你的关键词发给 ArXiv 网站，ArXiv 返回匹配的论文（标题、作者、摘要、链接）。

**核心代码逻辑：**
```python
# 1. 拼接搜索网址
url = f"http://export.arxiv.org/api/query?search_query=all:RAG&max_results=5"

# 2. 用 requests 库发送请求（就像浏览器访问网页）
response = requests.get(url)

# 3. ArXiv 返回的是 XML 格式（类似 HTML 网页代码），需要解析
root = ET.fromstring(response.text)  # ET = xml.etree.ElementTree

# 4. 从 XML 中提取标题、作者、摘要等信息
for entry in entries:
    title = entry.find("title").text    # 提取标题
    authors = ...                        # 提取作者
    summary = ...                        # 提取摘要

# 5. 拼成格式化的文本返回
return f"### {title}\n**作者**: {authors}\n**摘要**: {summary}"
```

### 🤖 工具2：summarize_paper — AI 总结论文

```
你点"AI总结" → 论文标题+摘要发给 LLM → LLM 返回中文总结
```

**通俗理解：** 论文摘要通常是英文的，而且写得很学术化。这个工具把标题和摘要发给大语言模型（就是你项目里用的 DeepSeek），让 AI 用通俗的中文告诉你这篇论文到底讲了什么。

**核心代码逻辑：**
```python
# 1. 构造提示词（告诉 AI 你想要什么）
prompt = f"请用通俗易懂的中文总结以下论文：\n标题：{title}\n摘要：{abstract}"

# 2. 调用大语言模型
response = completion(
    model="deepseek/deepseek-chat",   # 用哪个模型
    messages=[{"role": "user", "content": prompt}],  # 发送的消息
)

# 3. 返回 AI 的回答
return response.choices[0].message.content
```

### 🌐 工具3：translate_to_chinese — 翻译成中文

```
你点"翻译摘要" → 英文摘要发给 LLM → LLM 返回中文翻译
```

**通俗理解：** 和总结类似，但目的是忠实地翻译（不是总结），保持原文的所有信息。

**和总结的区别：**
- 总结 → "这篇论文提出了一个新方法，实验证明比旧方法好10%..."
- 翻译 → "我们提出了一种新的网络架构，称为Transformer，它完全基于注意力机制..."

## 怎么运行？

### 第一步：确保依赖都装好了

```bash
pip install streamlit litellm requests python-dotenv
```

### 第二步：确认 .env 文件配置正确

打开 `.env` 文件，确保有这两个配置：

```
DEEPSEEK_API_KEY=你的DeepSeek密钥
MODEL=deepseek/deepseek-chat
TAVILY_API_KEY=你的Tavily密钥（原有的Agent功能用的，Streamlit界面不需要）
```

### 第三步：启动

在项目根目录下运行：

```bash
streamlit run streamlit_app.py
```

浏览器会自动打开 `http://localhost:8501`，你就能看到界面了。

## Streamlit 界面是怎么工作的？

**Streamlit 的核心概念：每次你点击按钮，整个 Python 脚本从头到尾重新执行一遍。**

```python
# 这个变量每次都会重新生成
search_btn = st.button("搜索论文")  # 只有点击时才返回 True

# 但 session_state 里的东西会保留（像内存一样）
if "papers" not in st.session_state:
    st.session_state.papers = []   # 第一次运行时初始化

# 点击搜索按钮时
if search_btn and query:
    result = tools["search"](query)         # 调用搜索工具
    st.session_state.papers = parse_results(result)  # 保存结果

# 显示结果（每次运行都会显示）
for paper in st.session_state.papers:
    st.markdown(f"### {paper['title']}")   # 显示在页面上
```

**执行流程：**

```
1. 用户打开页面 → 脚本从头执行 → 初始化空状态 → 显示欢迎语
2. 用户输入关键词点搜索 → 脚本从头执行 → search_btn=True → 调用API → 保存结果
3. 用户点"AI总结" → 脚本从头执行 → 该按钮=True → 调用LLM → 显示总结
```

关键点：`st.session_state` 像一个**全局临时存储**，让数据在不同次执行之间不丢失。

## 你可能遇到的问题 & 解决方法

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 搜索返回空结果 | 关键词太生僻或网络问题 | 换个关键词试试 |
| AI总结/翻译报错 | API Key 没配或余额不足 | 检查 .env 里的 DEEPSEEK_API_KEY |
| 乱码 | 终端编码问题 | 不影响实际使用，只是在命令行测试时会乱码 |
| `ModuleNotFoundError` | 缺少依赖包 | `pip install streamlit` |
| ArXiv API 请求超时 | 网络慢 | 减少搜索数量，或换个时间段 |

## 总结：你学到了什么？

1. **如何写自定义工具** — 每个工具就是一个 Python 类，有 `name`、`inputs`、`__call__`方法
2. **如何调外部 API** — 用 `requests` 库发 HTTP 请求，解析返回的 XML/JSON 数据
3. **如何调大语言模型** — 用 `litellm` 库，构造提示词，获取回复
4. **如何写 Streamlit 界面** — 用 `st.button`、`st.text_input` 等组件，用 `st.session_state` 保存状态
5. **项目怎么串联起来** — 用户输入 → 工具处理 → 显示结果 → AI辅助理解
